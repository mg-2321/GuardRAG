                            WHAT'S INSIDE JSON FILES
═══════════════════════════════════════════════════════════════════════════════

Each file (nfcorpus_organized.json, fiqa_organized.json, scifact_organized.json)
contains:

✓ metadata
  ├─ corpus name
  ├─ document count
  ├─ poisoned count
  └─ description

✓ retriever_comparison
  ├─ bm25 _________________ Keyword-based (working ✓)
  ├─ dense ________________ Neural embedding (working ✓)
  ├─ hybrid_0.2 __________ 20% dense, 80% BM25 (working ✓)
  ├─ hybrid_0.5 __________ 50% dense, 50% BM25 (working ✓)
  ├─ hybrid_0.8 __________ 80% dense, 20% BM25 (working ✓)
  └─ splade ______________ Sparse neural (ERROR ✗)
       │
       ├─ stage3_retrieval_evaluation
       │  ├─ poison_presence_in_top_k (top-1, top-3, top-5, top-10)
       │  ├─ rank_of_first_poisoned_document (average position)
       │  └─ rank_distribution_by_attack_family
       │     └─ families (12-13 attack types per corpus)
       │        └─ Each family has:
       │           ├─ count (# of poisoned docs)
       │           ├─ mean (average rank)
       │           ├─ median (middle rank)
       │           ├─ min/max (best/worst positions)
       │           ├─ std_dev (variance)
       │           └─ distribution (frequency at each rank)
       │
       └─ stage5_packing_evaluation
          ├─ position_survival (%)
          │  ├─ start (injected at document start)
          │  ├─ early_mid (early-middle section)
          │  ├─ mid (middle section)
          │  ├─ late_mid (late-middle section)
          │  └─ end (document end)
          │
          └─ structural_carrier_survival (%)
             ├─ caption (in captions)
             ├─ code (in code blocks)
             ├─ table_caption (in table descriptions)
             ├─ citation (in citations)
             ├─ html_comment (in HTML comments)
             └─ json_meta (in JSON metadata)
